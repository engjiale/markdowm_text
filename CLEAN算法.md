### CLEAN算法

**面临的核心痛点**：在射电干涉测量中，由于设备故障、粗糙网格或超大间距等实际操作限制，干涉仪基线在 $u,v$ 平面上的分布往往是不规则的 。这种测量间隙会导致标准傅里叶反演生成的合成图（“脏图”）受到严重干扰，其中充满了合成波束（“脏束”）产生的显著且广泛的旁瓣效应。

**传统处理方法的局限**：在处理这类非规则间距数据时，早期的模型拟合法容易变得过于复杂，而插值法和最大熵方法在计算成本或普适性上存在明显短板。

**引入先验知识**：真实的高分辨率天空亮度分布图在大部分区域通常是空白的（低于仪器灵敏度），仅在极小部分区域存在显著辐射。

**提出的核心方案**：论文详细介绍并确立了 CLEAN 程序的地位。这是一种在地图平面上运行的迭代技术，它巧妙利用脏束的已知形状，将真实的辐射结构与虚假的旁瓣干扰区分开来。

**算法的适用边界**：当目标源过于复杂（所需参数多于测量次数）或数据充满噪声时，CLEAN 会在早期中断，无法彻底清除干扰 。此外，由于实际基线分布的三维特性（即存在 $w$ 项），合成波束的形状会随着偏离视场中心而改变，这限制了算法在大视场下的有效范围。

------

### 什么是 CLEAN 算法？

**定义** CLEAN 算法是一种用于射电天文图像处理的非线性、迭代式反褶积程序 。它的主要功能是直接在生成的图像（脏图）平面上，利用已知的仪器响应函数（脏束），将真实的天体信号与仪器观测不完美带来的伪影（旁瓣）剥离开来，从而“清洗”出接近真实情况的图像。

**运作原理（四个核心步骤）**

您可以将 CLEAN 算法的运行想象为一种“剥洋葱”式的不断识别与扣除的过程：

**第一步（初始化）**：通过傅里叶变换计算出包含旁瓣干扰的初始脏图以及对应的脏束。

**第二步（寻找并扣除最大值）**：在地图上寻找绝对值最大的点 $|I_o|$。认定此处为一个真实辐射源，并以该点为中心，在整张地图上减去一个按特定比例（环路增益 $\gamma$）缩放的脏束图案。

**第三步（循环迭代）**：在减去脏束后剩余的地图上重复寻找最大值并扣除的过程。当剩余的最大值低于背景噪声水平（即不再具有显著物理意义）时，停止迭代。

**第四步（图像重建）**：将前面步骤中每次减去的目标，以没有旁瓣的理想波束（洁净波束）的形式，重新加回到最后剩余的残差地图上，生成最终的高质量合成图。

**主要应用场景**

**处理非理想观测数据**：专用于解析因各种客观原因无法在 $u,v$ 平面形成规则测量网格的干涉仪数据，例如长基线测量或空间飞行器观测

**月掩星观测数据解析**：处理因月球掩蔽射电源而产生的一系列非规则位置角上的条带扫描数据。

**提升仪器运转效率**：通过该算法，可以用远少于常规要求的基线测量次数（和观测时间）获得特征基本正确的高分辨率合成图，大幅提高干涉望远镜的利用率。

**极化结构分析**：通过运行复数版本的程序或分别处理 Q 和 U 偏振地图，绘制射电源的极化结构。

### 算法解析

这份 MATLAB 代码展示了 CLEAN 算法在雷达信号处理（具体来说是距离-多普勒图，Range-Doppler Map, RDM）中的一个经典应用。这与之前天文学中处理不规则基线的场景有所不同，在这里，它主要被用作一种超分辨率（Super-resolution）技术，用来解决强目标的旁瓣掩盖弱目标的问题。

我来为您详细拆解这段代码的实现逻辑，并对比它与传统 FFT 的核心区别。

------

#### 一、 代码实现深度解析

这段 `clean_algorithm.m` 脚本完美映射了 CLEAN 算法的四个核心步骤。不同于天文学中复杂的仪器“脏束”，这里的“脏束”被简化成了雷达信号处理中理想的点目标响应函数（Point Spread Function, PSF）。

**1. 初始化与“脏束”（PSF）的构建**

在代码的第 19-27 行，算法首先构建了一个二维的点扩展函数（PSF）。

- **为什么用 Sinc 函数？** 在传统的基于 FFT 的雷达处理流程中，如果没有加窗（Windowing），一个理想的单点目标经过二维 FFT 后，在距离-多普勒图上的能量分布呈现为二维的 $sinc$ 函数形状（$sinc(x) = \frac{\sin(\pi x)}{\pi x}$）。这个自带无数旁瓣的 Sinc 函数，就是这里的“脏束”。
- 代码生成了一个指定大小（由 `psf_range` 和 `psf_dopp` 决定）的二维 Sinc 矩阵，并将其峰值归一化为 1。

**2. 寻找全局最大值（找目标）**

在第 35-42 行的 `for` 循环中，算法开始迭代。首先在当前的残差图（初始即为原始 RDM）中寻找绝对值最大的点 `[peak_val, peak_idx] = max(abs(residual(:)))`。

- 算法默认这个最大值就是一个真实存在的目标。
- **停止准则：** 如果找到的峰值已经小于原始 RDM 最大峰值的 5%（`peak_val < 0.05 * max(abs(RDM(:)))`），说明剩下的可能只是背景噪声，算法自动停止迭代。

**3. 记录目标参数**

在第 44-48 行，算法记录下这个目标的坐标（距离维索引 `r_peak`、多普勒维索引 `d_peak`）以及它的复数幅度。

- 注意这里使用了 `loop_gain`（环路增益，默认 0.9）。这意味着每次并不把目标能量 100% 减掉，而是减去 90%，这是一种防止过度减法和发散的安全机制。

**4. 减去脏束（消除旁瓣）**

在第 50-63 行，算法将刚才构建的二维 PSF 乘以目标的幅度（已乘增益），然后从残差图的对应位置上“挖掉”（减去）这部分能量。

- 这个操作不仅去除了主瓣能量，更重要的是**去除了该目标在周围产生的巨大旁瓣**。
- 减完之后，原本隐藏在这个强目标旁瓣下的小目标，就会在下一次迭代的残差图中显露出来。

####  二、 CLEAN 算法与传统距离维 FFT 的本质区别

传统雷达信号处理通常是做两次 FFT：一次距离维 FFT（快时间），一次多普勒维 FFT（慢时间），从而得到 RDM。这段代码中的 CLEAN 算法是在传统的二维 FFT **之后**进行的后处理。

它们的本质区别可以通过以下几个维度来理解：

| **比较维度**   | **传统距离-多普勒 FFT**                                      | **CLEAN 算法 (作为 FFT 的后处理)**                           |
| -------------- | ------------------------------------------------------------ | ------------------------------------------------------------ |
| **数学性质**   | **线性变换。** 只是将信号从时域转换到频域，信号的能量结构保持不变。 | **非线性迭代。** 是一种基于假设（目标是孤立点）的参数估计和反褶积过程。 |
| **分辨率限制** | **受瑞利极限（Rayleigh Criterion）限制。** 距离分辨率受限于信号带宽，多普勒分辨率受限于观测时间。靠得太近的两个目标会糊在一起。 | **突破瑞利极限（超分辨率）。** 通过数学手段强行剥离目标的响应函数，理论上可以分辨比传统 FFT 更近的目标。 |
| **旁瓣问题**   | **存在严重的旁瓣（能量泄漏）。** 强目标的旁瓣很容易淹没附近较弱的目标，导致漏警。通常需要通过“加窗”（如汉明窗）来压低旁瓣，但这会进一步导致主瓣变宽，分辨率下降。 | **主动消除旁瓣。** 不需要加窗损失分辨率。它通过减去理想 PSF 的方式，直接把旁瓣从图上“擦掉”，让弱目标水落石出。 |
| **输出结果**   | 是一张**连续的能量分布图**（矩阵），每个像素点都有值，包含目标、旁瓣和噪声。 | 是一系列**离散的点目标列表**（距离、速度、幅度）外加一张只剩噪声的底图（Residual）。 |
| **计算复杂度** | **极低。** FFT 算法非常成熟且高效，适合硬件底层实时处理。    | **较高。** 需要不断寻找峰值并进行矩阵减法，迭代次数越多越耗时。 |

**总结来说：**

传统的距离维 FFT 就像是用普通的相机拍了一张照片，如果有个很亮的光源（强目标），它周围会有一圈光晕（旁瓣），可能会遮住旁边暗淡的星星。而 CLEAN 算法就像是一个极具耐心的修图师，他知道光晕的形状规律（PSF），于是一点点把最亮光源的光晕完美擦除，让原本看不见的暗星显现出来。

#### 三、雷达clean算法参数说明

 一、 模板覆盖范围：`psf_range` 与 `psf_dopp`

这两个参数共同决定了每次迭代时，我们要从雷达图（RDM）上扣除的“脏束模板（PSF）”的二维物理大小。实际扣除的矩阵大小为 $(2 \times \text{psf\_range} + 1) \times (2 \times \text{psf\_dopp} + 1)$。

你可以把它们理解为“橡皮擦的尺寸”。

 1. 设置过小（例如 `psf = 1`，只覆盖核心 3x3 区域）

- **现象：严重的目标分裂（虚警爆表）。**
- **原理：** 真实的雷达目标能量分布是一个向外扩散的 Sinc 函数（主瓣+诸多旁瓣）。如果你的模板太小，算法只会把最中间高耸的主瓣尖端削掉，而把周围的旁瓣原封不动地留在了图上。
- **后果：** 在下一次迭代时，算法会误把这些没擦干净的“旁瓣残余”当成新的弱目标提出来，导致一个真实目标被强行分裂成了好几个假目标。

 2. 设置过大（例如 `psf = 15`，覆盖 31x31 区域）

- **现象：计算极度缓慢，且可能“误伤”无辜。**
- **原理：** 每次迭代都需要进行大矩阵的裁剪和减法运算，极其消耗 CPU 资源。更致命的是，理论上完美的 Sinc 模板在远离中心的区域，往往与雷达实际的物理响应不一致（因为天线误差、加窗效应等）。
- **后果：** 在极远处用不匹配的模板强行做减法，不仅毫无意义，反而可能因为模板误差在原本干净的背景上引入额外的数值波动，甚至把恰好处于该区域的、毫无关联的另一个弱目标的能量给破坏掉。

**💡 工程调参建议：**

这两个参数应该刚好覆盖目标的主瓣和**第一/第二对旁瓣**即可。通常在雷达处理中，设置为 `3` 到 `5`（即 7x7 或 11x11 的矩阵）是比较均衡的经验值。

------

 二、 扣除比例：环路增益 `loop_gain` ($\gamma$)

环路增益决定了每次找到峰值后，我们要减去该峰值能量的百分比（取值范围通常在 0 到 1 之间）。

你可以把它理解为“修图时的耐心程度”。

 1. 增益过大（接近 1.0）

- **现象：迭代极快，但容易“挖坑”和产生涟漪。**
- **原理：** 算法非常激进，企图一次性把目标 100% 连根拔起。但正如之前所说，理论模板永远无法 100% 贴合实际目标的形状（特别是在目标没有精确落在网格中心，即发生 Off-grid 效应时）。
- **后果：** 一次性减去 100% 的能量，极其容易在二维图上减出负值（挖出大坑），从而在目标周围激发出严重的人工数值振荡（Ringing）。这些振荡会在后续迭代中被当成新目标，依然会导致目标分裂。

 2. 增益过小（例如 0.1）

- **现象：极其稳定，提取质量极高，但速度慢得令人发指。**
- **原理：** 算法变得极其谨慎，像剥洋葱一样每次只轻轻刮掉峰值的 10%。这极大地缓冲了模板不匹配带来的误差，不会产生剧烈的数值振荡。
- **后果：** 原本 1 次就能（大致）扣除的目标，现在可能需要迭代 15 次才能把它降到背景噪声以下。如果图上有多个目标，计算时间会呈指数级上升，这在需要实时处理的雷达系统中往往是不可接受的。

**💡 工程调参建议：**

- 在射电天文学（对实时性要求不高，但对图像保真度要求极高）中，`loop_gain` 经常被设得很低（如 0.1 ~ 0.25）。
- 在雷达信号处理（需要在精度和实时性中取舍）中，通常设置为 **0.5 到 0.8**。既能有效防止一次性过度扣除带来的振荡，又能保证一个目标能在 2~4 次迭代内被清理干净。

| **参数调整方向**             | **优点**                         | **缺点**                                     | **适用场景**                         |
| ---------------------------- | -------------------------------- | -------------------------------------------- | ------------------------------------ |
| **增大 PSF 尺寸**            | 旁瓣清理更彻底，减少目标自身分裂 | 计算耗时增加，可能因边缘模型不匹配引入新误差 | 目标极强、旁瓣极宽，且算力充足       |
| **减小 PSF 尺寸**            | 计算速度极快                     | 留下大量旁瓣残渣，必定导致目标严重分裂       | 对计算时间有严苛限制的粗略预处理     |
| **增大 `loop_gain` (趋近1)** | 迭代次数少，收敛快               | 极易引起数值振荡，产生假目标                 | 模板与实际信号**完美匹配**的理想情况 |
| **减小 `loop_gain` (趋近0)** | 剥离过程平滑安全，极大地抑制虚警 | 耗费海量迭代次数，算力成本高昂               | 算力充沛、模板匹配度差、需要极高精度 |

#### 总结

这份代码之所以能从“抽出 8 个目标”进化到“精准抽出 3 个”，是因为它打出了一套组合拳：

1. **源头治理**：传入窗函数生成完美匹配的 2D 模板，几乎不留残余能量。
2. **逻辑兜底**：增加了邻域距离判断，即便由于网格不对齐产生了一点点残余尖峰，也会被当成“太靠近已探测目标”而被无情清零。
3. **动态止损**：自适应噪声门限让算法能在“真正没目标”时立刻停机，而不是盲目刮底噪。
