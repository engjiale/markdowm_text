# 13. 联合参数精修代码解析

这份代码实现的是**已知 $K=2$ 时的双目标联合距离精细化估计**，它不是一个独立替代 NOMP 的检测器，而是在 NOMP 给出两个距离初值之后，进一步把两个近距离目标作为一个耦合模型整体优化

核心入口是：

```matlab
eval_omp_close_pair_limit('Method', 'joint_lm')
```

调用链大致为：

```text
eval_omp_close_pair_limit.m
    -> nomp_range_refinement.m
    -> pairwise_varpro_grid_initializer(...)
    -> joint_varpro_range_refinement.m
```

其中：

- `nomp_range_refinement.m` 负责检测和生成主初值；
- `pairwise_varpro_grid_initializer` 负责生成一个近距离二原子 VarPro 备选初值；
- `joint_varpro_range_refinement.m` 负责真正的双目标联合 Variable-Projection LM 精修

当前实现服务于项目里的 LFM-PSF 距离模型：原子不是纯正弦 steering vector，而是经过 LFM 脉压、频域窗和 ROI 截取后的 PSF。这个差异会影响 Newton 导数、目标函数形状和论文原式的可直接复用程度

## 一、NOMP 与联合精修的关系

### 1. NOMP 的项目流程

项目里的正式 NOMP baseline 是：

```text
输入: y, 离散 PSF 字典 A, 距离网格 range_grid, K=2, 连续 PSF model

1. 在当前残差上做网格检测
   support_k = arg max_i |a_i^H residual|^2 / ||a_i||^2

2. 将网格点 range_grid(support_k) 作为新目标初值

3. 对新目标做 single refinement
   默认是有限差分 Newton；search 只是工程对照模式

4. 对当前已经检测到的所有目标联合 LS 重估复幅度 amps

5. 对已检测目标做 cyclic refinement
   每次固定其他目标，用排他残差更新当前目标距离

6. 每轮 cyclic 后再次联合 LS 更新 amps

7. 重复直到检测到固定 K=2 个目标
```

对应代码：

```matlab
% nomp_range_refinement.m
for active_k = 1:K
    residual = y - Phi * amps;
    support(active_k) = detect_grid_atom(residual, A, support(1:active_k - 1));
    ranges(active_k) = range_grid(support(active_k));

    for refine_idx = 1:cfg.SingleRefineIterations
        [ranges(active_k), single_refinement_score(active_k, refine_idx)] = ...
            refine_one_range(residual, ranges(active_k), model, cfg, ...
                             range_bounds, search_half_width);
    end

    Phi = build_atoms(model, ranges(1:active_k));
    amps = solve_amplitudes(y, Phi, cfg.Ridge);

    [ranges(1:active_k), amps, cyclic_diagnostics{active_k}] = ...
        cyclic_refine_ranges(y, ranges(1:active_k), amps, model, cfg, ...
                             range_bounds, search_half_width, ...
                             sprintf('NOMP k=%d', active_k));
end
```

NOMP 的优点是检测流程完整，结构贴近原始 Newtonized OMP 思路；局限是 close-pair 很近时，两个目标的 PSF 原子高度相干，逐目标 cyclic refinement 中的排他残差会强耦合，容易出现合并、偏移或被局部峰吸住

### 2. 联合参数精修的项目流程

`joint_lm` 的流程是：

```text
输入: y, NOMP 得到的两个初始距离, model

1. 先运行 sequential/newton NOMP
   得到 initial_ranges = [R1_init, R2_init]

2. 可选地运行 pairwise grid VarPro initializer
   得到一个近距离二原子备选初值

3. 以 NOMP 初值和 pairwise 初值为 base starts
   再按 SeparationScales 缩放间距，构造多个起点

4. 对每个起点执行 refine_one_start
   给定距离对 R=[R1,R2]，先 LS/ridge LS 消去复幅度 c

5. 外层只优化两个距离参数
   使用有限差分投影 Jacobian + 阻尼 GN/LM

6. 在中心/半间距坐标中更新
   center = (R1+R2)/2
   halfsep = (R2-R1)/2

7. 加入 ROI、有序距离、最小间距、最大步长、残差单调下降约束

8. 比较所有起点的最终 objective，选择残差最小者输出
```

调用位置在 `eval_omp_close_pair_limit.m` 的 `case 'joint_lm'` 分支：

```matlab
[initial_ranges, support, ~] = nomp_range_refinement( ...
    y, A, range_grid, 2, model, ...
    'InitializationMode', cfg.NompInitializationMode, ...
    'RefinementMode', cfg.NompRefinementMode, ...
    ...);

pairwise_initial_ranges = pairwise_varpro_grid_initializer( ...
    y, A, range_grid, initializer_max_separation, ...
    cfg.JointInitializerMaxConditionNumber, cfg.RefineRidge);

[refined_ranges, ~, joint_diag] = joint_varpro_range_refinement( ...
    y, initial_ranges, model, ...
    'AlternativeInitialRanges', pairwise_initial_ranges, ...
    ...);
```

这说明 `joint_lm` 的定位不是“重新检测目标”，而是“拿 NOMP/备选初值继续做双目标联合连续优化”。

### 3. 两者的关键差异

| 项目 | NOMP | 联合参数精修 `joint_lm` |
| --- | --- | --- |
| 入口 | `Method='nomp'` | `Method='joint_lm'` |
| 目标 | 检测 K 个目标并逐个精修距离 | 已知 K=2 时同时精修两个距离 |
| 初值 | 当前残差上逐次网格检测 | NOMP 初值 + 可选 pairwise grid VarPro 初值 |
| 更新方式 | 单目标 Newton/search，cyclic 逐个更新 | Variable Projection 消去幅度后，两个距离联合 LM 更新 |
| 幅度处理 | 每轮 refinement 后联合 LS 更新 `amps` | 每个距离对上都先 LS/ridge LS 求最优 `amps` |
| 适用场景 | 一般稀疏检测与连续精修 | close-pair 强相干区的后续局部精修 |
| 主要风险 | 近距离时逐目标排他残差强耦合 | 依赖初值，仍可能落入错误局部极小值 |
| 是否估计 K | 当前固定 K=2 | 当前固定 K=2 |
| 是否贝叶斯 | 否 | 否 |

一句话概括：

```text
NOMP 是“边检测边逐目标精修”；
joint_lm 是“用 NOMP 给初值，然后把两个近距离目标作为一个耦合模型整体优化”
```

## 二、联合参数精修的核心算法

### 1. 变量投影：消去复幅度

联合精修对应的原始问题可以写成：

$$
\min_{R_1,R_2,c_1,c_2}
\left\|y-\Phi(R_1,R_2)c\right\|_2^2
$$

其中：

- $R_1,R_2$ 是两个目标距离；
- **$c$ 是一个 $2 \times 1$ 的列向量**：$c = \begin{bmatrix} c_1 \\ c_2 \end{bmatrix}$，$c_1,c_2$ 是两个复幅度；
- $\Phi(R_1,R_2)=[a(R_1),a(R_2)]$ 是连续 PSF 原子矩阵

对于给定距离 $R=[R_1,R_2]^T$，幅度是线性参数，可以直接最小二乘求出：

$$
\hat c(R)=\arg\min_c
\left\|y-\Phi(R)c\right\|_2^2
$$

于是外层只优化两个实数距离：

$$
F(R)=
\left\|y-\Phi(R)\hat c(R)\right\|_2^2
$$

代码对应：

```matlab
function [amps, residual, objective, Phi] = variable_projection(y, ranges, model, ridge)
Phi = build_atoms(model, ranges);
amps = solve_linear_coefficients(Phi, y, ridge);
residual = y - Phi * amps;
objective = real(residual' * residual);
end
```

注意：当 `Ridge = 0` 时，这是标准 LS 消元；当前正式评估里通常使用很小的 `Ridge`，因此实际是正则化 LS 消元，而不是严格 Moore-Penrose 投影。

### 2. 多起点：降低初值敏感性

close-pair 的剖面目标函数可能存在多个局部极小值。如果 NOMP 初值已经落在合成峰附近，单一起点的 LM 很容易沿着错误盆地下降。

因此代码先构造多个起点：

```matlab
if isempty(alternative_ranges)
    base_ranges = initial_ranges.';
    base_source = {'nomp'};
else
    base_ranges = [initial_ranges.'; alternative_ranges];
    base_source = [{'nomp'}, alternative_labels(:).'];
end

separation_scales = unique([1, cfg.SeparationScales(:).'], 'stable');
```

每个 base start 都会转换成中心/间距形式，再按 `SeparationScales` 缩放初始间距：

```matlab
initial_center = mean(constrained_base);
initial_separation = diff(constrained_base);

separation = max(cfg.MinSeparationM, ...
                 initial_separation * separation_scales(scale_idx));

start_ranges(start_idx, :) = enforce_pair_constraints( ...
    initial_center + 0.5 * separation * [-1, 1], ...
    range_bounds, cfg.MinSeparationM).';
```

最后对所有起点分别优化，选 objective 最小者：

```matlab
for start_idx = 1:num_starts
    start_results{start_idx} = refine_one_start( ...
        y, start_ranges(start_idx, :).', model, cfg, range_bounds);
    selection_objective(start_idx) = start_results{start_idx}.objective;
end

[~, best_start] = min(selection_objective);
best = start_results{best_start};
refined_ranges = best.ranges;
```

这不是全局优化保证，只是一个工程上很有效的局部稳健化策略

### 3. 投影 Jacobian：只优化距离方向

在 `refine_one_start` 中，先对两个连续 PSF 原子分别做有限差分：

```matlab
atom_derivatives = finite_difference_atoms( ...
    model, ranges, cfg.DerivativeStepM, range_bounds);
```

由于幅度会在每个距离对上重新 LS 求解，距离导数中有一部分变化可以被幅度更新吸收，代码用 Kaufman 型 Variable-Projection Jacobian 近似去掉这部分：

```matlab
scaled_derivatives = atom_derivatives .* reshape(amps, 1, []);

projected_derivatives = scaled_derivatives - ...
    Phi * solve_linear_coefficients(Phi, scaled_derivatives, cfg.Ridge);
```

直观理解：

- `atom_derivatives .* amps` 表示距离变化对模型信号 $\Phi c$ 的一阶影响；
- `Phi * solve_linear_coefficients(...)` 表示这部分变化中可以由重新调整幅度解释的成分；
- 两者相减后，留下真正需要靠移动距离来解释的方向

`Ridge = 0` 时更接近标准正交投影形式；`Ridge > 0` 时是正则化 LS 算子对应的投影近似

### 4. 中心/半间距坐标

两个距离直接优化时，$R_1,R_2$ 的更新容易混合“整体平移”和“改变间距”，代码改用：

$$
R_1 = c-s,\qquad R_2 = c+s.
$$

其中 $c$ 是中心距离，$s$ 是半间距。代码中的映射为：

```matlab
% r1 = center-halfsep, r2 = center+halfsep.
coordinate_map = [1 -1; 1 1];
model_jacobian = projected_derivatives * coordinate_map;
```

这个变换的作用是让步长限制和最小间距约束更自然，尤其便于区分：

- 两个目标一起平移；
- 两个目标互相靠近或远离

它不会凭空增加物理可辨识信息，也不保证改善问题条件数；真正的信息量仍由 PSF、SNR、相位、目标间距和模型匹配程度决定

### 5. 阻尼 Gauss-Newton / LM

代码用投影 Jacobian 构造近似 Hessian 与梯度右端：

```matlab
hessian = real(model_jacobian' * model_jacobian);
hessian = 0.5 * (hessian + hessian.');
gradient_rhs = real(model_jacobian' * residual);
```

然后做 LM 更新：

```matlab
effective_damping = min(cfg.MaxDamping, damping * coherence_factor);
lm_matrix = hessian + effective_damping * diag(hessian_diagonal);
coordinate_step = lm_matrix \ gradient_rhs;
```

这里的 LM 步带有几个保护：

```matlab
if largest_step > cfg.MaxStepM
    coordinate_step = coordinate_step * (cfg.MaxStepM / largest_step);
end

candidate_ranges = enforce_pair_constraints( ...
    candidate_coordinates(1) + candidate_coordinates(2) * [-1; 1], ...
    range_bounds, cfg.MinSeparationM);

if candidate_objective < objective - required_decrease
    ranges = candidate_ranges;
    amps = candidate_amps;
    residual = candidate_residual;
    objective = candidate_objective;
    damping = max(eps, damping * cfg.DampingDecrease);
    accepted = true;
else
    damping = min(cfg.MaxDamping, damping * cfg.DampingIncrease);
end
```

也就是说，候选步必须让残差目标下降才会被接受；否则增大阻尼并重试

### 6. 相干性感知阻尼

当两个目标距离很近时，两列原子高度相似，代码记录：

$$
\rho =
\frac{|a(R_1)^H a(R_2)|}{\|a(R_1)\|\|a(R_2)\|}
$$

并使用两原子 Gram 矩阵条件数：

$$
\kappa=\frac{1+\rho}{1-\rho}
$$

代码：

```matlab
function [rho, kappa] = pair_coherence(Phi)
rho = abs(Phi(:, 1)' * Phi(:, 2)) / ...
    ((norm(Phi(:, 1)) + eps) * (norm(Phi(:, 2)) + eps));
rho = min(max(real(rho), 0), 1);
kappa = (1 + rho) / max(1 - rho, eps);
end
```

然后用 $\kappa$ 放大 LM 阻尼：

```matlab
coherence_factor = min(cfg.MaxCoherenceDamping, ...
    max(1, kappa / cfg.CoherenceKappaReference));
```

需要注意：

- 这里的 `kappa` 是两原子 Gram 条件数；
- 原子矩阵 $\Phi$ 本身的条件数是 $\sqrt{\kappa}$；
- 相干性阻尼是稳定性启发式，不是性能提升的唯一原因，也不是理论可分辨性保证

## 三、论文方法如何用到项目中

当前 `joint_lm` 不是逐式复现某一篇论文，而是把几类成熟思想组合到当前 LFM-PSF 雷达距离模型中

| 思想来源 | 在项目中的使用方式 | 对应代码位置 |
| --- | --- | --- |
| NOMP | 负责 sequential detection、single refinement、cyclic refinement 和 LS 幅度重估，提供两个主初值 | `nomp_range_refinement.m` |
| Variable Projection | 对给定距离对先用 LS/ridge LS 消去复幅度，外层只优化距离 | `variable_projection(...)` |
| Gauss-Newton / LM | 用投影 Jacobian 近似 Hessian，加入阻尼、步长限制和残差单调接受 | `refine_one_start(...)` |
| Parameter-Refined OMP | 提供“OMP/NOMP 检测后继续精修连续参数”的雷达稀疏估计思路 | `joint_lm` 整体设计 |
| QNOMP / quasi-Newton 类方法 | 作为后续进一步做联合 quasi-Newton 精修的参考 | 当前代码不是逐式复现 |
| VALSE/MVALSE/SBL | 可作为未来未知 K、贝叶斯 gridless 估计路线 | 当前没有采用 |

更具体地说：

1. NOMP 论文中的思想被用在前端检测上：先从残差中检测一个新原子，再做 Newton refinement，再做 cyclic refinement
2. Variable Projection 被用在后端联合精修上：把复幅度当作可解析消去的线性参数
3. LM 被用来稳定求解非线性距离参数，尤其是在 close-pair 高相干区域
4. pairwise grid VarPro 初值是项目里的工程增强，用来补偿 NOMP 初值可能被合成峰误导的问题
5. 相干性阻尼是项目针对双目标强相干场景加的稳定性启发式

## 四、与论文原式的主要不同

### 1. 原子模型不同

原始 NOMP 通常面对的是复正弦模型：

$$
y = \sum_k g_k a(\omega_k) + z
$$

当前项目使用的是 LFM-PSF 距离模型：

$$
y = \sum_k c_k a(R_k) + n
$$

其中 $a(R)$ 是经过 LFM 脉压、频域窗、DFT/ROI 等处理后的 PSF 原子，因此项目不能直接照搬纯正弦模型下的解析 Newton 公式，而是使用有限差分构造导数

### 2. Newton refinement 的实现不同

NOMP 原论文里可以针对正弦原子写出解析一阶、二阶导数，项目中：

```matlab
first_derivative = (atom_plus - atom_minus) / (2 * step_m);
second_derivative = (atom_plus - 2 * atom + atom_minus) / (step_m^2);
```

也就是说，当前 Newton 是基于连续 PSF 生成器的有限差分近似

### 3. 没有启用 CFAR 自动停止

原始 NOMP 通常包含检测阈值或 CFAR 风格停止准则，用来判断是否继续添加新频率/目标。当前 close-pair 实验中：

```text
K_true = 2
K_est  = 2
```

也就是固定目标数，`joint_lm` 也只处理两个目标，不负责估计 $K$

### 4. 从逐目标 refinement 扩展为双目标联合 refinement

原始 NOMP 的 refinement 主要是：

```text
检测一个目标 -> 单目标 Newton refinement -> cyclic refinement
```

当前 `joint_lm` 在 NOMP 之后增加：

```text
[R1, R2] 同时优化
```

这是项目的关键改造，它不是 NOMP 原式的一部分，而是为了解决 close-pair 下两个目标互相耦合、逐目标更新不稳定的问题

### 5. 加入多起点与工程约束

当前代码还加入了：

- NOMP 初值 + pairwise grid VarPro 初值；
- `SeparationScales` 多间距缩放起点；
- ROI 边界；
- 有序距离；
- 最小间距；
- 最大步长；
- 残差单调下降接受；
- 高相干条件数放大阻尼。

这些都属于项目级工程增强，不是 NOMP 原论文标准公式

### 6. 不是贝叶斯估计

虽然文档里提到了 VALSE/MVALSE/SBL 等路线，但当前 `joint_lm` 不是贝叶斯方法：

- 不估计后验概率；
- 不估计目标数 $K$；
- 不更新稀疏先验或噪声精度；
- 不做模型证据比较。

如果后续要处理未知 $K$，应该另行考虑 CFAR + BIC/MDL、VALSE/MVALSE 或 SBL

## 五、诊断信息怎么看

`joint_varpro_range_refinement.m` 会输出 `diagnostics`，其中比较重要的是：

```text
initial_ranges
nomp_initial_ranges
alternative_initial_ranges
start_ranges
start_source
selected_start
selected_start_source
selected_start_initial_relative_residual
nomp_initial_relative_residual
alternative_initial_relative_residual
relative_residual_history
coherence_history
condition_number_history
atom_matrix_condition_number_history
damping_history
coordinate_step_history
accepted_history
final_relative_residual
final_coherence
final_condition_number
```

分析失败样本时，建议优先看：

1. `selected_start_source`：最终是 NOMP 初值胜出，还是 pairwise grid VarPro 初值胜出。
2. `nomp_initial_relative_residual` 与 `alternative_initial_relative_residual`：初值质量是否已经明显不同。
3. `final_coherence` / `final_condition_number`：是否已经进入极高相干病态区。
4. `accepted_history`：LM 步是否频繁被拒绝。
5. `coordinate_step_history`：是中心方向在动，还是半间距方向在动。
6. `final_relative_residual`：失败是因为局部极小值，还是两个目标物理上已经接近不可辨。

## 六、当前结论的正确表述

建议在报告或论文草稿中这样表述：

```text
本文在距离维 NOMP 的基础上，引入已知 K=2 场景下的双目标联合 Variable-Projection LM 精修。
该方法首先使用 sequential/newton NOMP 给出目标距离初值，并可结合近距离二原子 VarPro 网格初值；
随后通过变量投影消去复幅度，在中心/半间距坐标下联合优化两个连续距离参数。
为提高 close-pair 强相干区域的数值稳健性，算法加入了有限差分投影 Jacobian、
相干性条件数感知阻尼、步长限制、边界/最小间距约束、残差单调接受和多起点择优
```

不要写成：

```text
本文完整复现了 QNOMP / VALSE / 贝叶斯 gridless 方法
```

也不要把开发期小样本结果直接写成最终结论。当前更稳妥的说法是：

```text
开发期小样本实验显示，双目标联合 VarPro-LM 精修在 close-pair 场景下能明显改善 NOMP 的局部精修结果；
但最终性能仍需同 seed、同容差、同 trials 的正式 Monte Carlo，以及相位/幅度敏感性和失败样本诊断来确认
```

## 七、一句话总结

NOMP 解决的是“怎么从残差里逐个找目标并连续精修”；联合参数精修解决的是“当两个目标已经很近、逐个精修开始互相干扰时，如何把它们作为一个双目标模型一起优化”

当前项目把 NOMP、Variable Projection 和 LM 的思想接到了 LFM-PSF 雷达测距模型上，但为了适配 PSF 原子和强相干 close-pair 场景，做了有限差分、多起点、边界约束、残差单调接受和相干性阻尼等工程化改造

## 八、与 PROMP 路线的区别

PROMP 可以作为“OMP 检测后做参数精修”的文献依据之一，但当前 `joint_varpro_range_refinement.m` 不是 PROMP 的复刻。两者最核心的差异是：PROMP 把幅度和位置都放进一个实数状态向量里做全参数 Gauss-Newton；当前项目则用 Variable Projection 先消去复幅度，外层只优化两个距离。

| 对比维度 | PROMP | 当前 `joint_lm` |
| --- | --- | --- |
| 数据表示 | 将复回波拆成实部/虚部，拼接为实数向量 | 保持复数向量 `y`，直接用 MATLAB 复数 LS |
| 状态变量 | 每个目标包含幅度实部、幅度虚部和位置参数 | 外层只保留两个距离 `ranges` |
| 幅度处理 | 幅度作为状态变量参与 Gauss-Newton 更新 | 幅度由 LS/ridge LS 在每个距离候选处解析重估 |
| 雅可比 | 对全参数状态构造实数 Jacobian | 对距离原子做有限差分，再做投影 Jacobian |
| 坐标 | 原论文中的物理位置坐标，例如旋转目标成像的 $(x,y)$ | 距离中心与半间距坐标 |
| 定位 | 参数精修 OMP 的一种全参数实现路线 | NOMP 初值后的双目标 VarPro-LM 后处理 |

因此更稳妥的表述是：

```text
PROMP 提供了“OMP 检测后继续做连续参数精修”的雷达稀疏估计语境；
当前 joint_lm 采用的是另一条变量分离路线：复幅度由最小二乘消去，距离由投影 Jacobian 和 LM 联合精修。
```

不要写成“当前方法一定比 PROMP 更优”。当前方法确实维度更低，也更贴合本项目固定 $K=2$ 的距离 close-pair 场景；但 PROMP 与 `joint_lm` 面向的前向模型、状态参数和实验任务不同，优劣需要在统一数据、统一指标和统一计算预算下单独比较。
